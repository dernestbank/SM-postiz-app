# OpenClaw integration (QuantGist)

This repo is the **git source of truth** for QuantGist Postiz Docker + app code. OpenClaw scripts use a stable symlink so paths do not break when you push from GitHub.

## Paths

| Role | Path |
|------|------|
| Git repo (edit & push) | `/Volumes/ExtHDD/github/QG-ecosystem/SM-postiz-app` |
| OpenClaw runtime | `~/.openclaw/infra/postiz-quantgist` → symlink to this repo |
| API key (never commit) | `~/.openclaw/credentials/postiz-quantgist-api-key.txt` |
| Ops docs | `~/.openclaw/workspace/workspace_QuantGist/ops/POSTIZ.md` |

## Public URLs

| Item | Value |
|------|--------|
| UI / signup | https://smm.quantgist.com/auth |
| API base (CLI) | `POSTIZ_API_URL=https://smm.quantgist.com/api` |
| Local Docker | http://localhost:4007 |

## Symlink (one-time)

```bash
# Stop stack from old upstream clone if it was a separate directory
docker compose -f ~/.openclaw/infra/postiz-quantgist/docker-compose.yaml down

# Replace orphaned gitroomhq/postiz-docker-compose clone with your repo
mv ~/.openclaw/infra/postiz-quantgist ~/.openclaw/infra/postiz-quantgist.upstream.bak
ln -s /Volumes/ExtHDD/github/QG-ecosystem/SM-postiz-app ~/.openclaw/infra/postiz-quantgist

# Copy secrets if you only had them in the backup tree
cp -n ~/.openclaw/infra/postiz-quantgist.upstream.bak/postiz.env ~/.openclaw/infra/postiz-quantgist/postiz.env 2>/dev/null || true
chmod 600 ~/.openclaw/infra/postiz-quantgist/postiz.env

# Start from symlinked repo (compose project name: postiz-quantgist)
docker compose -f ~/.openclaw/infra/postiz-quantgist/docker-compose.yaml up -d
```

`docker-compose.yaml` sets `name: postiz-quantgist` so named volumes stay attached when the compose file moves from the old clone to this repo.

## Local secrets

- `postiz.env` — generated by `scripts/render-postiz-env.sh` from `sm_credential.md`; listed in `.gitignore`
- Regenerate OAuth env: `bash scripts/render-postiz-env.sh` then `docker compose down && docker compose up -d`
- **Do not** set `JWT_SECRET` in `docker-compose.yaml` — it overrides `postiz.env` and breaks login cookies.
- **After postgres volume wipe:** re-register or sign in at https://smm.quantgist.com/auth, then **Settings → API** → copy org key into `~/.openclaw/credentials/postiz-quantgist-api-key.txt` (chmod 600). Old keys return 401. Re-connect all channels in the UI.
